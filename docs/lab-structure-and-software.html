Lab Structure and Software

R and RStudio

If you are brand new to using R/Rstudio, you will need to download and install both. Once everything is installed and ready to use, I recommend starting with R for Data Science. It’s freely available and decently comprehensive - you can work your way through the chapters and will learn most of the basics you will need for coding in R. This is something you should work on in the first four months of your program if you have little to no prior experience.

I do not recommend using AI to provide coding/anaylsis help unless you are very confident in your coding abilities already. AI should be viewed as a tool, and it can be useful in some contexts, but only if you can evaluate it critically. You are responsible for the code that you produce, which means you need to be able to understand and justify everything that you do.

Github

Please note - even if you are experienced using Github, you will need to follow some of the instructions below to link your projects and repositories to our lab organization

Much of our lab materials will be organized on Github. There is a bit of a learning curve associated with this, but ultimately it is a platform that will serve you well beyond your time in our lab, and familiarity with it will make you a stronger quantitative scientist.

Below you will find step-by-step instructions to help you set up an account in our Github organization and start your first project. If you are brand new to using Github, start here…

Making an Account

Navigate to https://github.com/ and create an account, if you don’t already have one. Once you’ve done this, please reach out to Allison and share your username, so you can be added to our Lab’s Github organization page.

Repositories and Projects

For each chapter of your thesis, you will create one repository to keep all the data, scripts for analysis, and other relevant materials in one place. To do so, navigate to our lab’s page, click on the Repositories tab on the top banner, then click the green New Repository button in the right hand corner. Once you are on the “Create a new repository” page, there are a few fields that require your attention:

  • ensure that under “Owner” it says “Binley-Lab”
  • give your repository a somewhat informative name (e.g., avian solar-energy interactions)
  • next to “Choose visibility” select “private”
  • next to “Start with a template” select “Binley-Lab/repo-template”
  • set “Add README” to “On”
  • next to “add .gitignore” select “R” from the drop-down menu
  • click “create repository”

You should now have a repository set up on your account that is linked to our organization, and already set up using the repo-template I’ve designed to keep your files organized (more on this later). Next, we need to ensure that this repository is linked to the appropriate projects. In general, each of your repositories will be linked to two Github “projects”: one that is part of a larger suite of lab projects, and one that is specific to your thesis. The former will already exist, whereas the latter is something you will create for yourself.

To add a repository to an existing lab project: Only do this step once you have discussed with Allison which project your work belongs in. Navigate to your repository and click on “Projects” on the banner above. Click on “Link a project”, then select the appropriate project from the drop-down menu.

To create your own thesis project and add your repositories: From the main Binley-Lab page, navigate to the Projects tab. Click on the green “New Project” button, then select the “Team planning” template. Next, name your project a combination of your username and the type of thesis you are doing (e.g., @adbinley’s PhD thesis). Click on the “Select Repository” drop down menu and select the repository that you created to link that repository to your new thesis project. Note that when you create new repositories to add to your thesis (e.g., a PhD thesis will have multiple repositories, one for each chapter), you will need to manually link these repositories to your existing thesis project. Click “Create Project”.

Reference Management

Unless otherwise discussed, all lab members will use Zotero for reference and citation management. You can download Zotero for free here. If you are unfamiliar with Zotero, there are some useful tutorials available online. I also recommend using the Chrome or Firefox plug-in to make it easier to save papers. Please let me know if you use up the free storage space and we can upgrade it. Using reference management serves several purposes:

  1. It will save you many, many hours reformatting manuscripts to fit different journal requirements
  2. It provides a paper organization/file storage system that most likely beats whatever system you can set up manually
  3. We can create shared folders when collaborating on different projects, so that all collaborators have access to the same papers

If you are having trouble getting started with Zotero, you can reach out to the UM Library or we can set up a meeting to go through it.
